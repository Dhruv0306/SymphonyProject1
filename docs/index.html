Troubleshooting

Common Issues

Connection Error

If you see "Cannot connect to the server. Please ensure the backend is running.":

ERR_NETWORK: Cannot connect to the server
  • Check if the backend server is running at http://localhost:8000
  • Verify your internet connection
  • Ensure no firewall is blocking the connection

File Size Error

When uploading large files:

413 Request Entity Too Large: Image file is too large
  • Ensure each file is under 200MB
  • Compress large images before uploading
  • For batch processing, check total batch size

File Type Error

When uploading unsupported files:

415 Unsupported Media Type: Invalid file type
  • Check file extension (must be .jpg, .jpeg, or .png)
  • Ensure file is not corrupted
  • Convert images to supported formats

URL Processing Error

When using image URLs:

Invalid or inaccessible image URL
  • Verify URL points directly to an image file
  • Check if URL is publicly accessible
  • Ensure URL is properly encoded

API Reference

API Endpoints

The Logo Detection System provides RESTful API endpoints for both single image and batch processing:

Single Image Detection

POST /api/check-logo/single/

Request Format

Request
Content-Type: multipart/form-data

Parameters:
- file: Image file (optional)
- image_path: Image URL (optional)

# Note: Provide either file or image_path, not both

Success Response

Response (200 OK)
{
    "Image_Path_or_URL": "example.jpg",
    "Is_Valid": "Valid",
    "Error": null
}

Error Response

Response (400 Bad Request)
{
    "Image_Path_or_URL": "example.jpg",
    "Is_Valid": "Invalid",
    "Error": "Failed to process image"
}

Batch Processing

POST /api/check-logo/batch/

Request Format

Request
Content-Type: multipart/form-data

Parameters:
- files: Multiple image files (optional)
- paths: Semicolon-separated URLs (optional)

# Note: Provide either files or paths, not both

Success Response

Response (200 OK)
[
    {
        "Image_Path_or_URL": "image1.jpg",
        "Is_Valid": "Valid",
        "Error": null
    },
    {
        "Image_Path_or_URL": "image2.jpg",
        "Is_Valid": "Invalid",
        "Error": "No logo detected"
    }
]

Batch Statistics

GET /check-logo/batch/getCount

Response Format

{
    "valid": 5,
    "invalid": 3,
    "total": 8
}

Important Notes

  • All endpoints return HTTP 200 for successful requests
  • File size limit: 200MB per file
  • Supported formats: JPG, JPEG, PNG
  • URLs must be publicly accessible
  • Batch processing supports up to 50 images

Frequently Asked Questions

What image formats are supported?

The system supports PNG, JPG, and JPEG formats.

Is there a file size limit?

Yes, each file must be under 200MB.

Can I process multiple URLs at once?

Yes, in batch mode you can enter multiple URLs, one per line.

What happens if the server is down?

You'll see a connection error message. Try again later or contact support.
